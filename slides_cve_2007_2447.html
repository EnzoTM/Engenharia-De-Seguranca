Seção 2 · Fundamentação teórica

Samba, SMB e CIFS

O Samba é uma implementação livre dos protocolos SMB (Server Message Block) e CIFS (Common Internet File System) para Unix/Linux, uma ponte de interoperabilidade que permite compartilhar arquivos e impressoras com clientes Windows.

Daemon

O processo principal é o smbd, responsável por atender as sessões SMB.

Portas

TCP/139 NetBIOS Session Service
TCP/445 SMB direto sobre TCP

Por que importa

SMB é onipresente em redes corporativas, falhas nele são especialmente críticas.

Seção 2 · Fundamentação teórica

A diretiva username map script

Em smb.conf, essa opção delega o mapeamento de nomes de usuário a um script externo: o cliente fornece um nome, o Samba invoca o script com esse nome, e o script devolve o usuário local correspondente. A intenção era flexibilidade administrativa.

A origem do problema

O nome de usuário é entrada controlada pelo cliente

Esse nome chegava ao script sem sanitização adequada. Ao montar a linha de comando para invocar o script, metacaracteres de shell presentes no nome podiam ser interpretados pelo interpretador de comandos.

Seção 2 · Fundamentação teórica

CVE-2007-2447, fluxo da exploração

1Cliente SMB inicia sessão e fornece um nome de usuário na mensagem Session Setup AndX Request.
2Com username map script ativo, o Samba invoca o script externo passando esse nome.
3A montagem da linha de comando não sanitiza metacaracteres de shell.
4Caracteres como acentos graves ` ` fazem o shell interpretar o conteúdo como comando.
5O comando é executado no contexto do smbd, que roda como root.

Dois pontos que tornam a falha grave

Ponto 01

Pré-autenticação

O nome de usuário é processado antes da autenticação. O atacante não precisa de credencial alguma, o que diferencia radicalmente esta falha de "senha fraca".

Ponto 02

Execução como root

O smbd roda com privilégios máximos, então o comando injetado herda esse contexto, impacto máximo sobre o sistema.

Impacto na tríade CIA

Confidencialidade

Afetada diretamente. O atacante leu o /etc/shadow e obteve as senhas de todos os usuários sem jamais fornecer uma senha válida.

Seção 4 · Impacto e análise de protocolo

Por que não é "senha fraca"

A falha ocorre antes da autenticação. É uma vulnerabilidade de injeção de comando (command injection), tratamento inseguro de entrada.

  • No fluxo SMB, o campo de nome de usuário é processado na negociação de sessão, antes de qualquer verificação de senha.
  • A prova está no pacote: o campo Account carrega o comando shell em vez de um nome.
  • Não há troca de senha em momento algum, credenciais são irrelevantes para a exploração.
● CVE-2007-2447 · SambaInjeção de comando, não senha fraca
Seção 4 · Impacto e análise de protocolo

Hashes MD5-crypt e quebra de senha

O problema

Hash antigo e barato

O Metasploitable2 usa $1$ (MD5-crypt), algoritmo computacionalmente barato. O John the Ripper começa pelo single crack mode, testa o próprio nome de usuário como senha. Contas como msfadmin:msfadmin caem instantaneamente.

O contraste

Hashing moderno

Com $6$ (SHA-512) ou $y$ (yescrypt), cada tentativa de quebra é ordens de magnitude mais cara, o que torna o ataque inviável na prática.

Seção 5 · Defesa em profundidade e conclusão

Por que a mitigação funcionou

A defesa não bloqueia o pacote, o campo Account ainda carrega a injeção. O que muda é para onde essa entrada vai. É eliminar a causa-raiz, não contornar o sintoma.

Antes · vulnerável

username map script

Invoca um script externo passando a entrada do cliente, que chega a um interpretador de comandos.

Depois · corrigido

username map (estático)

Lê um arquivo de mapeamento fixo. A entrada do cliente nunca chega a um shell. O serviço continua ativo.

Seção 5 · Defesa em profundidade e conclusão

Camadas adicionais de defesa

01

Senhas fortes

Senhas longas e aleatórias tornam a quebra inviável, mesmo que um hash vaze.

02

Hashing moderno

Trocar $1$ por $6$ ou $y$ eleva o custo de cada tentativa.

03

Menor privilégio

Se o smbd não rodasse como root, o comando não acessaria o /etc/shadow.

04

Restrição de rede

hosts allow/deny e firewall limitam quem acessa as portas SMB.

05

Monitoramento / IDS

Nomes de usuário com metacaracteres em pacotes SMB são assinatura de exploração.

06

Auditoria de config

Revisar o smb.conf em busca de diretivas perigosas alimentadas pelo cliente.
